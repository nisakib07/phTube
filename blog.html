Scope of var, let and const

Var, let and const are three methods used in JavaScript to declare a variable. Though the initial objective of these three is same, their working procedure and scope are slight different. Here, we are going to find out the differences regarding the scope of them.

Var : Scope of var variable is functional scope. The value of this variable can be updated and re-declared. This type of declarations are hoisted to the top of that variable or in a global scope which makes it possible to use this variable before initialization though the value will be undefined.

Let : The scope of let variable is block scope. This type of variables can be declared without initialization. Though the value of this type variable can be updated, it can not be re-declared. It is not hoisted like `var`, so using the variable before initialization is impossible.

Const : Like the 'let' variable, this too is a block scoped variable. But the difference is 'const' variables can neither be updated or re-declared. It is not also possible to declare this type of variable without initialization. Hoisting to the global space does not happen in this case.

Use cases of null and undefined

'null' and 'undefined' are two primitive type data value in JavaScript. While 'undefined' refers the absence of any value, 'null' defines the intentional absence of any value. The use cases of these two are described below:

'undefined' :

  • - if any variable is not initialized
  • - if nothing returned from the function
  • - if any value of parameter not given in a function
  • - if there is nothing written on the right side of return, it will be undefined
  • - accessing a property if that is not available in the object
  • - if an index is not present in an array
  • - if the value of something is set as undefined

'null' :

  • - to indicate a falsy value
  • - if the value of a variable is set as null
  • - if the value of an object property is cleared
